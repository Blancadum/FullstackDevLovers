## What Each Repte Asks For

### Repte 1: Project Proposal
**Question:** Why this project? Who needs it?
**Answer:** 7 documented sections covering business case, competition, costs, GDPR

### Repte 2: Technical Design
**Question:** How will you build it? With what tools?
**Answer:** Technology stack, development tools, prototypes (3-5 screens)

### Repte 3: Investor Pitch
**Question:** Why should we invest in your project?
**Answer:** 1-minute elevator pitch + peer feedback on others' ideas

### Repte 4: Team Organization
**Question:** How will your team work together?
**Answer:** Team composition, communication plan, roles, chosen project

### Repte 5: SCRUM Framework
**Question:** How will you use agile methodologies?
**Answer:** Agile methodology comparison, SCRUM planning, Taiga setup, EPICs & User Stories

### Repte 6: First Sprint
**Question:** Build a working part of the project
**Answer:** Sprint tasks + weekly updates + working code + demo videos

### Repte 7: Second Sprint
**Question:** Build a bigger working part
**Answer:** Improved sprint tasks + weekly updates + more working features + demo videos

### Repte 8: Final Delivery
**Question:** Complete the entire project
**Answer:** Final features + full documentation + professional presentation + project evaluation

## SCRUM Terminology Quick Reference

| Term | Meaning | When Used |
|------|---------|-----------|
| **Epic** | Large feature group | Planning phase |
| **User Story (US)** | Single user-facing functionality | Sprint planning |
| **Sprint** | 1-4 week development iteration | Active work |
| **Task** | Technical work unit (part of US) | Daily work |
| **Product Owner** | Prioritizes requirements | All phases |
| **Scrum Master** | Facilitates process, removes blockers | All phases |
| **Development Team** | Implements features | Sprint execution |
| **Product Backlog** | All project requirements | Planning |
| **Sprint Backlog** | Sprint-specific work | Active sprint |
| **Sprint Increment** | Functional product part | Sprint end |
| **Daily Standup** | Brief status sync (adapted to weekly) | Sprint execution |
| **Sprint Review** | Demonstrate completed work | Sprint end |
| **Sprint Retro** | Reflect on improvements | Sprint end |

## Evaluation Focus Areas

### Individual Challenges (Reptes 1-3)
- **Research Quality:** Market analysis, competitive landscape
- **Technical Accuracy:** Sound technology choices
- **Presentation:** Clear, professional documentation
- **Feasibility:** Realistic project scope for course

### Team Challenges (Reptes 4-5)
- **Organization:** Clear processes and roles
- **Communication:** Documented coordination
- **SCRUM Adoption:** Proper methodology implementation

### Development Challenges (Reptes 6-8)
- **Functionality:** Working features at each sprint
- **Code Quality:** Clean, maintainable code
- **Documentation:** Complete technical docs
- **Teamwork:** Balanced contribution and communication
- **Agile Practice:** Proper SCRUM execution

## Common Mistakes to Avoid

### Documentation
- ❌ Missing evaluation criteria sections
- ❌ Incomplete GDPR/data protection discussion
- ❌ Vague functionality descriptions
- ✓ Use provided templates and examples

### Prototyping
- ❌ Less than 3 screens
- ❌ No navigation between screens
- ❌ Low visual coherence
- ✓ Aim for 4-5 professional screens

### Team Work
- ❌ Unclear communication plan
- ❌ Unbalanced task distribution
- ❌ Missing weekly SCRUM minutes
- ✓ Document everything in Taiga Wiki

### SCRUM Execution
- ❌ No working code at sprint end
- ❌ Poorly defined User Stories
- ❌ Insufficient task breakdown
- ✓ Every sprint needs functional increment

### Video Submissions
- ❌ Videos over time limits
- ❌ No clear demonstration of features
- ❌ Missing team member explanations
- ✓ Professional, clear demonstrations
