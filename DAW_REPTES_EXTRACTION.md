## REPTE 5: Agile Methodologies and SCRUM Planning (Group)
**Document:** `DAW_0616_Repte5_Ejemplo.docx`
**Type:** Group Challenge
**Submission Status:** Part of coursework

### Learning Outcomes
- Implementation of agile methodologies (SCRUM)
- Project planning with iterative and incremental approach

### Task Structure (Scoring)

**A. Agile Methodologies and Project Management Tools (4 points)**

#### Questions to Answer
1. Comparative analysis of Agile vs. Traditional methodologies
2. Research and list current agile project management tools

#### Agile vs. Traditional Methodologies Comparison

**Agile Methodologies (e.g., SCRUM):**
- **Focus:** Iterative and incremental development
- **Structure:** Project divided into short cycles (sprints)
- **Output:** Functional product at end of each sprint

**Advantages of Agile:**
- High flexibility and adaptability
- Continuous stakeholder involvement
- Early problem detection
- Frequent functional software delivery

**Disadvantages of Agile:**
- Requires high team and client commitment
- Harder to estimate total cost and duration
- Challenging for teams without Agile experience

**Traditional Methodologies (Waterfall, RUP, Spiral):**
- **Structure:** Linear and sequential process
- **Approach:** Each phase completed before next begins
- **Characteristic:** More rigid but structured

**Advantages of Traditional:**
- Clear milestones and documentation
- Easier initial planning
- Suitable for stable requirement projects

**Disadvantages of Traditional:**
- Difficult and costly to adapt changes
- Testing occurs late (bugs found late)
- Less client interaction during development

**When to Use:**
- **Agile:** Dynamic projects, web/mobile apps, changing requirements
- **Traditional:** Projects with stable requirements, predictable scope

#### Agile Project Management Tools
- **Taiga:** Open-source, SCRUM/Kanban, epics, user stories, sprints, visual management
- **Jira:** Enterprise-level, deep customization, reporting, integration
- **Trello:** Visual cards/boards, small teams, simple Kanban
- **Asana:** Tasks, timelines, workflows, Agile practices
- **ClickUp:** Flexible workspace, sprint planning, AI assistance
- **Monday.com:** Visual project management, automation

**Tool Selection Rationale:** Improve transparency, collaboration, and control for quick functional increment delivery

**B. Project Planning with Agile Methodologies (SCRUM) (6 points)**

#### Components to Define
1. **EPICs:** Major features/functionality groups
2. **User Stories (US):** Individual work units describing functionality from user perspective
3. **Sprints:** Iteration cycles (3 sprints planned for this course)
4. **Sprint Definition:** Specify which US are included in each sprint
5. **SCRUM Roles:** Product Owner, Scrum Master, Development Team

#### SCRUM Workflow Overview
- **Product Backlog:** All project requirements/features
- **Sprint Planning:** Select US for sprint
- **Sprint:** Development iteration (typically 1-4 weeks)
- **Daily Standup:** Brief team sync (adapted to weekly in this course)
- **Sprint Review:** Demonstrate completed work
- **Sprint Retrospective:** Reflect on improvements
- **Sprint Increment:** Functional product piece
